# Glossary

| Term | Meaning |
| --- | --- |
| Accept | Parser action that finishes parsing successfully. |
| Action | User C code attached to a grammar rule. |
| Conflict | A parser state where more than one action is possible for the same lookahead. |
| Counterexample | A generated example explaining how a conflict can be reached. |
| Error recovery | Parser behavior that attempts to continue after a syntax error. |
| Grammar | A set of terminals, nonterminals, declarations, and rules describing a language. |
| IELR | LR table construction mode enabled with `%define lr.type ielr`. |
| Item | A grammar rule plus a marker showing parser progress through the right-hand side. |
| LALR(1) | Lookahead LR parser table construction using one lookahead token and merged compatible states. |
| Lexer | Code that converts input characters into parser tokens. |
| Location | Source position information carried by `YYLTYPE`. |
| Lookahead | The next token considered by the parser while choosing an action. |
| Nonterminal | A grammar symbol defined by rules. |
| Parameterized rule | An Lrama rule template that takes symbols as arguments. |
| Parser | Generated code that recognizes token sequences according to the grammar. |
| Precedence | Ordering used to resolve some shift/reduce conflicts. |
| Reduction | Parser action that replaces matched right-hand side symbols with a left-hand side nonterminal. |
| Semantic value | Data carried by a token or nonterminal through `YYSTYPE`. |
| Shift | Parser action that consumes the lookahead token and moves to a new state. |
| Skeleton | Template used to render generated parser output. |
| State | A parser automaton node containing LR items and transitions. |
| Syntax diagram | HTML visualization of grammar rules generated by `--diagram`. |
| Terminal | A token consumed by the parser. |
| Token | Symbol returned by `yylex`. |
